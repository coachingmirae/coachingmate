    파이프라인 실행이 failed로 표시될 때, 아래 순서로 원인을 확인하고 복구합니다.

            확인 항목
            점검 방법

            로그
            아래 실행 이력에서 failed 행의 보기를 클릭해 STDERR 확인
          

          

            GPT 최종 결과
            3번/4번 integrated_final JSON이 저장되어 있는지 확인
          

          

            템플릿
            DOCX 템플릿 파일이 templates 폴더에 존재하는지 확인
          

          

            생성 파일
            generated 폴더에 1번 PDF, 2번 DOCX, 3번/4번 DOCX가 있는지 확인
          

        
      

    


    

      
2. 자주 발생하는 실패 유형

      

        
          

            오류 유형
            복구 방법
          

        
        
          

            FORCE_FAIL 테스트
            의도적 실패입니다. 정상 실행 버튼으로 다시 실행하면 됩니다.
          

          

            GPT 결과 없음
            GPT 결과 JSON 저장 영역에서 integrated_final 최종 결과를 저장하세요.
          

          

            템플릿 파일 없음
            templates/docx/coach_guides 폴더의 current 템플릿 파일명을 확인하세요.
          

          

            파일 없음
            1번 PDF 또는 2번 DOCX가 generated 폴더에 있는지 확인 후 다시 등록하세요.
          

          

            Supabase 로그인 실패
